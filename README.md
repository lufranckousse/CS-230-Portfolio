# CS-230-Portfolio

Module Eight Journal Reflection
Client and Requirements

The client, The Gaming Room, wanted to expand their Android game Draw It or Lose It into a web-based application that can run across multiple platforms. The system needed to support multiple players, teams, and game sessions while maintaining performance and consistency. It also required enforcing unique names and using a single instance of the game service to avoid conflicts.

What I Did Well

I applied object-oriented design and used design patterns like Singleton effectively. This helped keep the system organized and ensured consistency across the application. I also made sure the design met the requirements, especially for handling unique names and multiple sessions.

Helpful Design Process

The design document helped me plan before coding. Breaking the system into requirements and architecture made it easier to understand how everything connects and reduced confusion during development.

What I Would Improve

I would improve the scalability and security sections by going deeper into cloud deployment, load balancing, and more advanced authentication methods like token-based authentication.

Interpreting User Needs

I focused on making sure the system supports multiple users and runs reliably across platforms. Considering user needs is important because it directly impacts how effective the system is.

Design Approach

I used object-oriented principles and design patterns to structure the system. In the future, I would also focus more on cloud-based design, microservices, and scalability from the start.
